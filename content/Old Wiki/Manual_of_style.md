This guide is intended to inform users of how pages should be designed
and formatted. This page is not a strict list of rules, but contains the
preferred formatting of pages of various types.

## General formatting {#general_formatting}

An article should not lead with a header. This is unnecessary given that
headers are already present in article starts that has the name. Instead
a brief explanation of the item (what kind it is, eg component or
holdable item, as well as where it is obtained or how it is created)
should be provided.

If there is an applicable template for the item, eg. componentinfo or
item, it should occupy the first line. These templates will
automatically orient themselves into the top right of the page if placed
in this location, making it look clean and easily readable. All of a
template\'s required areas should be filled in before publishing the
article. Optional areas are to be filled in if they are relevant to the
subject of the article.

The main body of a page should be in standard paragraph, not heading or
subheading format. Subheadings are to be used sparingly and only when a
standard heading or paragraph text would not be applicable. Headers
should separate distinct sections (eg, function and gallery), and should
be used only for that purpose.

When creating a new page, its name should be in sentence format (eg,
\"This is a sentence\") rather than other cases (THIS IS A SENTENCE,
This Is a Sentence, etc.). Pages that already have fully capitalized
names are \"grandfathered\" and allowed to stay. If an existing page has
the same name but is referring to a different subject, a link should be
added to the start explaining that there are multiple possible uses, and
whichever page is created second should have its name altered to reflect
what it is referring to (eg, Subject (descriptor)).

Casual language is acceptable so long as it is sparingly used and does
not interfere with the information in the article. This wiki is written
in English found not inside the United States, for example armour
instead of armor.

As is to be expected, all [wiki
rules](Wiki_Rules "wiki rules"){.wikilink} are to be followed with no
exception.

## Component articles {#component_articles}

Component articles should contain the {{ComponentInfo}} template. All
applicable information should be filled out, and an image of the
component in question if available.

After the template a brief explanation explaining if it is a component
machine object, manifest, or alien component should be added.

Next, a section (separated by a header) should explain the function of
the item. Go into as much detail as is possible.

Finally, if more images are available a gallery section should be added
containing the remaining images. If any images are needed, an ambox
template should be used to request images.

For reference, a good example of a properly formatted component article
can be seen [here](Seeker_(manifest) "here"){.wikilink}.

## Item articles {#item_articles}

Item articles are formatted similarly to a component article, instead
using the item template rather than the componentinfo template.
Obtainment should be clarified within the article.

A good example of a properly formatted item article can be found
[here](content/Old%20Wiki/Flintlock.md "here"){.wikilink}.
